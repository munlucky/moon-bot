## Git 워크플로우

> 브랜치 명명, 커밋 메시지, PR 규칙을 명시하면 에이전트가 그대로 따릅니다.

### 브랜치 명명 규칙
```
feature/{기능명}     # 새 기능
fix/{이슈번호}       # 버그 수정
refactor/{대상}      # 리팩토링
chore/{작업}         # 설정, 의존성 등
```

### 커밋 메시지 형식
```
[타입]: 간결한 설명

예시:
feat: 배치 실행 API 추가
fix: 날짜 포맷 변환 오류 수정
refactor: 사용자 조회 로직 분리
```

**규칙:**
- 이모지, 특수문자 제외
- 한글 또는 영문 통일
- 50자 이내 권장


### 경로 템플릿

| 문서 | 경로 패턴 |

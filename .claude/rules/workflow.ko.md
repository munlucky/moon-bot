# 작업 실행 방식

- 가능한 경우 실제 작업(파일 읽기/편집, 검증 실행)을 우선합니다.
- **자동 작업 분석**: 사용자 요청이 코드 작업(기능 추가/변경, 버그 수정, 리팩터링 등)이면 즉시 `/moonshot-orchestrator` 스킬을 실행합니다.
  - 단순 질문, 정보 조회, 읽기/설명만 필요한 작업은 제외합니다.
  - PM 오케스트레이터가 작업 유형/복잡도/필요 에이전트를 결정해 최적 체인을 실행합니다.
  - 워크플로우 상세: `.claude/skills/moonshot-orchestrator/SKILL.md`
- 정보가 부족하면 질문하거나 명시된 저위험 가정으로 진행합니다.
- 복잡한 작업은 plan -> implement -> verify -> summarize 순서를 따릅니다.
